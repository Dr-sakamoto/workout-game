# 筋トレクエスト（workout-game）

筋トレ・食事の記録が、そのままアバター（ドット絵ボクサー）の経験値になる
ゲーミフィケーション・フィットネスアプリ。
**Habitica の中毒性のある育成ループ × Punch Club のドット絵ボクサー**がコンセプト。

### 設計原則（DESIGN.md §0）

1. **装備はない** — 筋肉アプリ。見た目の魅力は鍛えた肉体そのものだけから来る。
2. **最初はショボい** — 鍛えて変わる「差分」が報酬。先に完成形は見せない。
3. **ライバルは自分** — ランキングは作らない。比較対象は常に過去の自分。

- 製品設計（収益化含む）: [DESIGN.md](./DESIGN.md)
- これは設計を検証するための **Web版MVPプロトタイプ**（React + TypeScript + Vite）。
  ドメインロジックはフレームワーク非依存に分離してあり、React Native への移植を想定。

## 実装済み（MVP）

- **キャラメイク**: 身長・体重（BMI）でアバターの体格が決定。入力に応じてリアルタイムに見た目が変化
- **ドット絵ボクサー**: 体格（BMI）と筋肉発達（レベル/STR）から手続き的に生成。
  「太っただけ（寸胴・腹）」と「鍛えた（逆三角・肩・腹筋）」が別物に見える。装備品は一切なし
- **Before/After（ライバルは自分）**: 最初の自分と今の自分を並べて成長を可視化。自己ベスト更新の演出
- **トレーニング記録 → EXP**: ボリューム（重量×レップ×種目係数）/ METs ベースでEXP・ゴールド・ステータスを獲得
- **5ステータス**: STR / END / VIT / AGI / DEX を部位ごとに育成
- **食事記録（PFC）→ コンディション**: タンパク質充足度でトレEXPに±補正。ワンタップ記録対応
- **かんたん記録**: バーコードのない自炊・外食は「タンパク質の量感」「全体の量」の2タップでPFCをざっくり推定。g単位の詳細入力は折りたたみ（初心者は記録の習慣化が最優先、精密さは任意）
- **食事記録の修正・削除**: 「今日の食事」をタップして修正・削除。バーコード由来の記録はコミュニティDBの誤登録もその場で直せる
- **レベルアップ**: `level^1.5` カーブ、レベルアップ演出
- **デイリークエスト & ストリーク**: 連続記録・日次ミッション
- **ボス戦**: トレEXPがダメージ。敵を倒して進む中期目標（最終ボスは「過去の自分」）
- **実績（トロフィー）**: レベル/連続/総トレ/ボス撃破/見た目変化のマイルストーン収集
- **ショップ**: ゴールドの使い道。EXPブースト/ストリーク保護の消耗品（装備・スキンは売らない＝原則1）
- localStorage に永続化

## 開発

```bash
npm install
npm run dev      # 開発サーバ (http://localhost:5173)
npm test         # ドメインロジックのユニットテスト
npm run build    # 型チェック + 本番ビルド
```

## 構成

```
src/
  domain/        フレームワーク非依存のコアロジック（移植可能）
    exercises.ts   種目マスタ・ステータス定義
    expEngine.ts   EXP/ゴールド/ステータス計算（純関数）
    avatar.ts      レベルカーブ・見た目ティア
    physique.ts    身長+体重 → 体格 → アバター見た目パラメータ
    meals.ts       PFC → コンディション（EXP補正）
    quests.ts      デイリークエスト評価
  store/         zustand ストア（永続化・ループの統合）
  components/    画面・UI（PixelAvatar = ドット絵ボクサー生成）
```
