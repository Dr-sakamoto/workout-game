# 筋トレゲーミフィケーションアプリ 設計ドキュメント

> コンセプト: **「トレーニングと食事の記録が、そのままアバターの経験値になる」**
> 現実の努力 → RPG 的なキャラ育成 → 続けたくなる、というループを作る。

---

## 0. 設計原則(ブレない軸)

このアプリの体験を定義する3つの原則。機能の取捨選択はすべてここに従う。

1. **見た目の魅力は「肉体そのもの」だけから来る — 装備はない。**
   筋肉アプリに外付けの装備やスキンは筋違い。アバターを着飾る手段は用意しない。
   かっこよさは鍛えた肉体でのみ表現される。

2. **最初はショボい。鍛えて変わる「差分」こそが報酬。**
   初期アバターはあえて貧弱に作る。先に完成形を見せたら意味がない。
   「過去の自分との見た目の差」を実感させることがリテンションの核。
   → だからこそ Before/After の可視化に最大限投資する。

3. **ライバルは自分。他人との順位はつけない。**
   ランキング/リーダーボードは作らない。比較対象は常に「過去の自分」。
   交流(フレンド・協力)はするが、それは競争ではなく励まし・共闘のため。

## 1. コアコンセプトとループ

リアルな筋トレ・食事という「面倒で続かない行動」を、RPG のキャラ育成という
「ドーパミンが出る体験」に変換するのが本質。中心となるエンゲージメントループ:

```
 トレーニング/食事を記録
        ↓
 経験値(EXP)とステータスに変換
        ↓
 アバターがレベルアップ・見た目が変化・強くなる
        ↓
 過去の自分(Before)と比べて成長を実感する
        ↓
 クエスト/ボス/協力で「使う」場ができる
        ↓
 もっと記録したくなる ──┐
        ↑              │
        └──────────────┘
```

成功の鍵は **「記録の手間 < 得られる報酬感」** を常に保つこと。
記録 UI は 10 秒で終わる軽さを最優先する。

## 2. アバター育成システム(中核)

### 2.1 ステータス設計

現実のトレーニング部位を RPG ステータスにマッピングする。
これにより「バランス良く鍛える」動機が自然に生まれる。

| ステータス | 対応する現実の鍛錬 | 上がる種目例 |
|-----------|-------------------|-------------|
| STR(筋力) | 高重量・低レップ | ベンチ、デッドリフト、スクワット |
| END(持久力) | 有酸素・高レップ | ランニング、サイクリング、バーピー |
| VIT(体力/HP) | 脚・体幹・総量 | スクワット、プランク、総ボリューム |
| AGI(俊敏) | 自重・瞬発系 | 懸垂、ジャンプ、HIIT |
| DEX(技巧) | フォーム精度・継続性 | ストリーク、種目バリエーション |

各部位・ステータスが個別にレベルを持ち、**偏ると見た目もアンバランスになる**
(脚だけ細い、など)ので、総合的に育てたくなる。

### 2.2 経験値の変換ロジック

「筋トレ記録がそのまま経験値」を担保する、透明で説明可能な式。

**ウェイトトレーニング(ボリュームベース)**
```
セットEXP = 重量(kg) × レップ数 × 種目係数
種目EXP   = Σ(各セットEXP)
獲得EXP   = 種目EXP / 10   // スケール調整
```

**自重トレーニング**
```
セットEXP = 体重(kg) × 自重係数 × レップ数 × 種目係数
```

**有酸素運動(METs ベース)**
```
獲得EXP = METs × 時間(分) × 体重係数
```
METs は厚労省『身体活動のメッツ表』準拠の値をテーブルで持つ。

**種目係数**は危険・高負荷なコンパウンド種目ほど高く設定し、
「ちゃんと追い込む」ことを報酬で誘導する(例: デッドリフト 1.5、
アームカール 0.8)。

### 2.3 レベルカーブ

```
次レベル必要EXP = baseEXP × (level ^ 1.5)
```
序盤はサクサク上がり(継続フックを作る)、上級者は伸びにくくして
やり込み度を担保する。プレステージ(転生)で上級者の長期目標も用意。

### 2.4 見た目の成長(原則1・2の実装)

アバターの見た目は **2軸のパラメータだけ** で手続き的に決まる。装備品は介在しない。

- **体格(横幅)** = 身長＋体重(BMI)。キャラメイク時に決まる初期値。
- **筋肉(肩幅・腕の太さ・胸/腹の陰影・僧帽筋)** = 鍛錬度(レベル・STR)。
  トレーニングでのみ育つ。**ここが見た目の魅力そのもの。**

初期アバターはあえて貧弱(撫で肩・細腕・陰影なし)に作る。
鍛えるほど肩が盛り上がり、腕が太くなり、腹筋・胸筋の陰影が出る。
「数字」ではなく **肉体の変化** で成長を見せるのがリテンションの肝。

### 2.5 Before / After 可視化(原則2・3の実装)

キャラメイク時の体格を **スナップショットとして保存** し、
ホームで「最初の自分」と「今の自分」を常に並べて見せる。
比較対象は他人ではなく過去の自分 = **ライバルは自分**。

- 「最初はこんなにショボかった」という差分が最大のモチベーション。
- 成長タイムラプス/Before-After 画像は SNS で自然にシェアされる
  (= オーガニック獲得。広告費をかけずに伸びる導線)。

## 3. 食事記録の統合

食事は「もう一つの経験値ソース」かつ「ステータス補正」として組み込む。

### 3.1 PFC をゲーム内パラメータへ

| 栄養 | ゲーム内の意味 |
|------|---------------|
| タンパク質(P) | 筋肉の「回復力」「成長補正」。目標達成でトレEXPにボーナス |
| 脂質(F) | 「スタミナ/燃費」。過剰だと debuff |
| 炭水化物(C) | 「行動エネルギー」。トレ前の摂取でその日のEXPブースト |
| 総カロリー | 目標(増量/減量)に対する達成度メーター |

### 3.2 メカニクス

- **コンディション制**: 食事バランスが良い日はアバターが好調(EXP +x%)、
  乱れるとデバフ。リアルな栄養管理がそのままゲーム的損得になる。
- **タンパク質クエスト**: 「体重 × 1.6g 達成」で日次ボーナス。
- **入力の手間対策**: バーコード/写真AI解析/よく食べる物のワンタップ登録で
  記録コストを最小化(継続率に直結)。

## 4. ゲーミフィケーション要素

| 要素 | 内容 | 目的 |
|------|------|------|
| デイリークエスト | 「胸を鍛える」「P 100g」等 | 毎日開く動機 |
| ストリーク | 連続記録日数・炎アイコン | 損失回避で継続 |
| **自己ベスト(原則3)** | 種目別の最高重量/ボリューム、最高EXP日、最長ストリーク。更新で「自己ベスト更新!」演出 | **過去の自分に勝つ** |
| Before/After | 最初の自分と今の自分を並べて表示 | 成長実感・自分との比較 |
| ボス戦 | 月間チャレンジ(累計ボリュームでボスを削る) | 中期目標 |
| 実績/バッジ | 初100kg、30連続 等 | 達成収集欲 |
| フレンド(協力のみ) | 協力ボス、励まし、進捗の共有 | 共闘で継続(競争ではない) |
| シーズン制 | 3ヶ月ごとに報酬リセット | 再エンゲージ |

> **ランキング/リーダーボードは作らない(原則3)。** 他人と順位で比べると、
> 体格・経験差で勝てない初心者が即離脱する。比較対象は常に「過去の自分」。
> フレンド機能は competitive ではなく cooperative(協力ボス・励まし)に限定する。
>
> 交流系は **継続率を最も押し上げる** 一方で実装コストも高いので、MVP 後に実装(§7)。

## 5. 収益化デザイン(重点)

健康アプリは「無料で十分」と思われやすいので、**複数レイヤー**で組む。
P2W(課金で強くなる)はトレ記録の信頼性を壊すので**避ける**。
**装備/スキンのコスメ課金もしない(原則1)** ので、
収益は **利便性(サブスク)・コマース・成長の記録** で立てる。

### レイヤー1: フリーミアム・サブスク(本命)
**Premium(月額 ~600–980円 / 年額割引)**
- 詳細分析(部位別推移、PFCグラフ、1RM推定)
- AIコーチ(次の重量提案、メニュー自動生成、フォーム/栄養アドバイス)
- 無制限の記録履歴(無料は直近 N 週)
- **成長の記録**: アバターの Before/After タイムラプス書き出し、
  ボディ写真の経過管理、月次の成長レポート(原則2の体験を深掘り)
- 広告非表示

> 健康・フィットネス領域はサブスク受容性が高い。LTV の主軸。
> 装備課金を捨てた代わりに「自分の成長そのもの」を売る — 原則と矛盾しない。

### レイヤー2: コマース/アフィリエイト(原則と相性◎)
**装備で課金させない代わりに、現実の肉体強化を後押しする物販で稼ぐ。**
PFC 記録から **プロテイン・サプリ・ヘルシー食材を文脈提案**
(「タンパク質不足→おすすめプロテイン」は CVR が高い)。
現実の体を強くする物販はアプリの目的と一致し、ユーザーの不利益にならない。
### レイヤー3: 広告(無料ユーザー)
- バナー/インタースティシャル(控えめに)
- **リワード動画**: 視聴で「EXPブースト」「ストリーク復活」
  → ユーザーメリットがあり、体験を壊しにくい。

### レイヤー4: トレーナー・マーケットプレイス
パーソナルトレーナーがメニュー/コーチングを販売、運営は手数料(テイクレート)。
ユーザーのリアルなデータがマッチング精度を上げる。

### レイヤー5: B2B / 法人向け
- ジム・企業の健康経営向けに、会員エンゲージメント SaaS として提供。
- 企業対抗チャレンジ、福利厚生パッケージ。安定収益源。

### 課金導線の設計
- まず**無料で「育つ快感」を体験**させ、データが溜まった頃に
  「もっと詳しく/もっと自由に」で Premium へ。
- 賞金付きチャレンジ等は景表法・賭博規制に注意(無償提供 or 法務確認必須)。

## 6. 技術スタック(推奨)

| 領域 | 推奨 | 理由 |
|------|------|------|
| クライアント | React Native (Expo) / Flutter | iOS/Android 同時、課金SDK充実 |
| アバター描画 | 2D(Live2D / Rive)から開始 | 3D は重い。段階的に |
| バックエンド | Supabase or Firebase | 認証/DB/同期を高速に |
| サーバーロジック | Edge Functions(EXP計算・不正検知) | クライアント改ざん防止 |
| 課金 | RevenueCat | iOS/Android サブスク横断管理 |
| 分析 | Amplitude / PostHog | ループのファネル計測 |

> EXP 計算・自己ベスト判定は**必ずサーバー側**で行い、
> クライアント改ざんによるチート(課金の公平性・記録の信頼性)を防ぐ。

## 7. MVP スコープと優先順位

**Phase 1 — MVP(まず作る / 検証する仮説: 「育つと続くか」)**
1. ユーザー登録・アバター作成(身長＋体重で体格決定 / 初期スナップショット保存)
2. ワークアウト記録(種目DB + セット入力) → EXP/レベル
3. アバターのレベルアップと見た目変化(肉体の段階的変化)
4. Before/After 表示・自己ベスト(ライバルは自分)
5. 簡易食事記録(PFC手入力) + コンディション補正
6. デイリークエスト + ストリーク

**Phase 2 — リテンション強化**
7. 詳細分析・グラフ、ボス戦、実績バッジ
8. **Premium サブスク + RevenueCat 導入(収益化開始)**
9. 成長タイムラプス書き出し・ボディ写真の経過管理
10. 食事の写真/バーコード入力

**Phase 3 — グロース & マネタイズ拡張**
11. フレンド(協力ボス・励まし)※ランキングは作らない
12. コマース/アフィリエイト、トレーナーマーケット、リワード広告
13. B2B 提供

## 8. データモデル(概略)

```
User(id, name, height, weight, goal[bulk/cut/keep], created_at)
Avatar(user_id, level, total_exp, str, end, vit, agi, dex)   // 装備カラムなし(原則1)
StartSnapshot(user_id, height, weight, level, stats, created_at)  // Before表示用(原則2)
PersonalBest(user_id, exercise_id, best_volume, best_weight, best_day_exp, best_streak)  // ライバルは自分(原則3)
Exercise(id, name, category, target_stat, coefficient, mets)   // マスタ
WorkoutLog(id, user_id, date, exercise_id, sets[{weight,reps}], earned_exp)
MealLog(id, user_id, date, name, protein, fat, carb, calories, earned_exp)
DailyCondition(user_id, date, pfc_score, exp_modifier, streak_count)
Quest(id, type, condition, reward_exp, period)
Subscription(user_id, tier, status, renews_at)   // RevenueCat 同期
```

## 9. 主要 KPI

- **継続率**: D1 / D7 / D30 リテンション(最重要)
- **記録率**: DAU あたりの記録完了率(ループが回っているか)
- **課金転換**: 無料→Premium CVR、ARPPU、解約率
- **エンゲージ**: ストリーク中央値、クエスト達成率

> 北極星指標(North Star)候補: **「週3回以上記録する習慣ユーザー数」**。
> ここが伸びれば収益は後からついてくる。
